## Definition

An **Intent** is the structured description of what a principal declares it is about to do. It is passed as part of the `POST /oauth/2/token` request and embedded in the issued [Mandate](/concepts/mandate/).

An intent has three fields:

| Field | Required | Description |
|---|---|---|
| `action` | Yes | Verb describing the operation. Convention: `<domain>:<verb>` (e.g., `file:read`, `api:call`, `db:query`). |
| `resource` | Yes | Identifier of the target. Can be a URL pattern, a resource ID, or a glob (e.g., `/data/reports/*.csv`). |
| `context` | No | Arbitrary JSON object with additional context the policy may inspect. |

## Why it exists

Without a declared intent, a mandate is just a credential — it proves identity but not purpose. Intent is what makes Caracal's enforcement *pre-execution*: the agent must commit to what it will do before authority is granted. The policy evaluates the specific intent, not just "is this agent authenticated."

The gateway also checks that the actual outbound request matches the resource pattern in the mandate's intent. An agent cannot declare `intent.resource = /data/public/*` and then make a request to `/data/sensitive/keys.json`.

## How the gateway validates intent

After verifying the mandate's signature and TTL, the gateway checks that the actual request target matches `intent.resource`. The match uses the same glob semantics the STS used at issuance. If the request path does not match, the gateway returns `403 intent_mismatch` and records the event in the ledger.

## Examples

```typescript title="Declare intent in a run() call — TypeScript SDK"
import { CaracalClient } from '@caracalai/sdk'

const client = new CaracalClient({
  coordinatorUrl: process.env.CARACAL_COORDINATOR_URL,
  gatewayUrl: process.env.CARACAL_GATEWAY_URL,
})

const session = await client.run({
  intent: {
    action: 'api:call',
    resource: 'https://api.example.com/v1/reports/*',
    context: {
      // context is available to the policy as input.intent.context
      reportType: 'quarterly',
      requestedBy: 'analytics-pipeline',
    },
  },
})
```

## Constraints and limits

- An intent is immutable after issuance. The mandate embeds the intent at signing time.
- The `resource` field supports glob patterns (`*`, `**`). The gateway uses the same pattern at request time.
- The `action` field has no enforced format, but the `<domain>:<verb>` convention is strongly recommended for policy readability.
- `context` values are passed as-is to OPA. Keep context payloads small (under 4 KB); large contexts increase policy evaluation latency.
