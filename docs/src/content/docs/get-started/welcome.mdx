Caracal helps teams run AI agents in real environments without handing long-lived provider credentials to agent code. An agent asks Caracal for scoped authority, policy decides before the action happens, the Gateway or a verified service enforces the result, and audit records the evidence.

## Who Caracal is for

Caracal is built for teams that need to answer these questions:

- Which agent app was allowed to call this resource?
- Which policy allowed or denied the request?
- Which run, delegation edge, and resource were involved?
- Can access be revoked without restarting the whole platform?
- Can a child agent receive narrower permission without copying secrets?

Caracal is not an LLM framework, prompt router, agent scheduler, or provider SDK wrapper. It is the authority and audit boundary between agent code and protected tools, APIs, providers, or data.

## The default path

```text
agent app -> SDK or caracal run -> Gateway -> protected resource
                         |             |
                         |             +-> action-result audit
                         +-> STS policy decision + mandate
```

In the default path:

1. A confidential agent app exchanges its app secret for a short-lived Caracal access token.
2. The STS evaluates the active policy set before signing the token.
3. The agent calls the Gateway with the mandate and resource identifier.
4. The Gateway verifies the mandate, checks replay and revocation, routes to the upstream, and emits action-result audit.
5. Operators use the Console **audit** and **explain** flows to understand the decision.

Use this path first. Move to connector-verified services only when you own the protected service boundary and can preserve mandate verification, revocation, replay protection where needed, and result audit there.

## What you need to know first

You can complete onboarding without learning every internal object. The useful first vocabulary is small:

| Term | Meaning |
| --- | --- |
| Agent app | The workload identity registered in Caracal. |
| Agent session | One tracked execution of that workload. |
| Mandate | The short-lived signed authority Caracal issues after policy allows access. |
| Resource | The protected tool, API, provider, or data target. |
| Delegated permission | A narrowed authority path from one agent session to another. |

The reference pages use precise terms such as principal, execution session, delegation edge, policy set, and audit event. [Key Ideas at a Glance](./key-ideas/) maps those terms when you need them.
