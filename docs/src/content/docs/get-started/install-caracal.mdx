Install the released runtime when you want to run Caracal as a product. The `caracal` CLI owns local runtime lifecycle, workload execution, upgrades, and the local web development launcher. The packaged web console is a deployable production service in Compose and Helm; it owns product management: zones, applications, resources, providers, policies, sessions, audit, diagnostics, and Control workflows.

## Install the Runtime CLI

Release assets include platform-specific `caracal-runtime-*` archives, `manifest.json`, and `SHA256SUMS`. The `install.sh` installer installs the runtime-only CLI from those assets; it does not install source-tree development tooling. You can use the installer or download the archive for your platform from the [Caracal releases](https://github.com/Garudex-Labs/caracal/releases), verify it, and place the `caracal` binary on your `PATH`.

To install with the release script on Linux or macOS:

```sh
curl -fsSL https://raw.githubusercontent.com/Garudex-Labs/caracal/main/install.sh | sh
```

To verify a download yourself, or to verify archives and container images fetched directly, follow [Verify a Release](/security/verify-releases/).

### Linux and macOS

```sh
gh release download vYYYY.MM.DD \
  --repo Garudex-Labs/caracal \
  --pattern 'caracal-runtime-linux-amd64-*.tar.gz' \
  --pattern SHA256SUMS
sha256sum --check SHA256SUMS
tar -xzf caracal-runtime-linux-amd64-vYYYY.MM.DD.tar.gz
install -m 0755 caracal ~/.local/bin/caracal
```

Choose the `linux-arm64`, `darwin-amd64`, or `darwin-arm64` archive when that matches your machine. Ensure the destination directory is on your `PATH`.

### Windows

```powershell
gh release download vYYYY.MM.DD `
  --repo Garudex-Labs/caracal `
  --pattern 'caracal-runtime-windows-amd64-*.zip' `
  --pattern SHA256SUMS
Get-FileHash .\caracal-runtime-windows-amd64-vYYYY.MM.DD.zip -Algorithm SHA256
Expand-Archive .\caracal-runtime-windows-amd64-vYYYY.MM.DD.zip -DestinationPath $env:LOCALAPPDATA\Programs\caracal
```

Compare the PowerShell hash with the matching entry in `SHA256SUMS`, then add `%LOCALAPPDATA%\Programs\caracal` to the user `Path` and open a new shell.

## Verify Installation

```sh
caracal --version
caracal status --help
caracal web --help
```

On Windows PowerShell:

```powershell
caracal --version
caracal status --help
caracal web --help
```

Use `caracal` for local lifecycle, workload execution, upgrades, and the local web development launcher. Use the packaged web console at `http://localhost:3001` after `caracal up` for the standard local product-management path.

## Verify Docker

Running the open-source stack locally requires Docker 24+ with the Compose v2 plugin:

```sh
docker compose version
```

On Windows PowerShell:

```powershell
docker compose version
```

The local stack includes PostgreSQL, Redis, and the packaged web console. You do not need to install those databases separately for Get Started.

## Start the Local Stack

```sh
caracal up
caracal status --ready
```

On Windows PowerShell:

```powershell
caracal up
caracal status --ready
```

After readiness succeeds, open [http://localhost:3001](http://localhost:3001). Compose maps the host address `127.0.0.1:3001` to the web BFF container on port `3002`. Use `caracal web` only when you need the local development launcher backed by Vite.

## Platform Notes

| Platform | Architectures | Notes |
| --- | --- | --- |
| Linux | `amd64`, `arm64` | Requires `curl` or `wget`, `tar`, `sha256sum` or `shasum`, and GitHub CLI `gh` for provenance verification. |
| macOS | `amd64`, `arm64` | Requires GitHub CLI `gh`; if Gatekeeper quarantines the binary, remove quarantine from the installed file. |
| Windows | `amd64` | Requires GitHub CLI `gh`; open a new shell after updating the user `Path`. |
