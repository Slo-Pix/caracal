Use this path after installing the released `caracal` and `caracal-console` tools. It starts the local stack and hands product setup to the Console.

## Start the stack

```bash
caracal up
caracal status --ready
```

`caracal up` starts the local services. `caracal status --ready` waits for the stack to become usable. If readiness fails, run:

```bash
caracal status --ready --json
```

The JSON output shows which service is not ready. Fix runtime issues before creating product objects.

## Open the Console

```bash
caracal console
```

The Console is the product-management surface. Inside it, use arrow keys or number shortcuts to navigate and press `?` for in-app help.

For the first run, choose **guided setup**. The wizard walks through:

| Step | What you choose |
| --- | --- |
| Zone | Create or select the isolation boundary. |
| Agent app | Create or select the workload identity. |
| Provider | Create or select the credential source when the resource needs one. |
| Resource | Create or select the protected API, service, provider target, or tool. |
| Access policy | Create and activate the starter least-privilege policy. |
| Review | Confirm the objects, runtime profile, secret file, and first-success commands. |

Keep advanced options closed unless you need a custom resource identifier, first request path, custom profile path, skipped policy activation, or explicit overwrite behavior.

## Save the generated runtime profile

Guided setup can write the runtime profile and secret file for you. Enable **write profile files** when you want the Console to create them on the current machine. If you leave file writing off, copy the generated TOML profile and one-time app client secret from the result view into the paths the Console shows.

The profile is the source of runtime configuration for:

- `CARACAL_CONFIG=... caracal run -- <command>`
- `Caracal.connect()` in TypeScript and Python.
- `caracal.Connect()` in Go.

Always generate a real profile from the objects you just created instead of hand-authoring runtime configuration.

## Stop the stack

```bash
caracal down
```

Use `caracal purge` only when you intentionally want to remove local containers, volumes, config, runtime state, and caches.
