---
title: "Quickstart: Run Caracal Locally"
description: Bring up the complete Caracal authority plane and launch the Console.
pageType: workflow
---

The quickest local path is: install dependencies, start the stack, verify readiness, then use the Console for product setup.

This source-tree flow is for contributors and local evaluation. If you installed the released binaries, use the same commands without the `pnpm` prefix: `caracal up`, `caracal status --ready`, and `caracal console`.

```bash
corepack enable
corepack prepare pnpm@11.1.1 --activate
pnpm install --frozen-lockfile --prefer-offline
pnpm caracal up
pnpm caracal status --ready
pnpm caracal-console
```

Use the Console to create the first zone, application, resource, and policy. When you need credential injection for a local process, provide the values produced by the Console through the same env/secret-file interface used by release deployments:

```bash
printf '%s\n' '<client-secret>' > "$XDG_RUNTIME_DIR/caracal-client-secret"
chmod 600 "$XDG_RUNTIME_DIR/caracal-client-secret"
cat > "$XDG_RUNTIME_DIR/caracal-credentials.json" <<'JSON'
[{ "env": "RESOURCE_TOKEN", "resource": "resource://my-api" }]
JSON
export CARACAL_STS_URL="http://localhost:8080"
export CARACAL_ZONE_ID="<zone-id>"
export CARACAL_APPLICATION_ID="<application-id>"
export CARACAL_APP_CLIENT_SECRET_FILE="$XDG_RUNTIME_DIR/caracal-client-secret"
export CARACAL_RUN_CREDENTIALS_FILE="$XDG_RUNTIME_DIR/caracal-credentials.json"
```

Then run protected workloads with:

```bash
pnpm caracal run -- <command>
```

Stop the stack with:

```bash
pnpm caracal down
```

Use [First Integration](./first-integration/) for the full protected HTTP request and audit verification flow.
