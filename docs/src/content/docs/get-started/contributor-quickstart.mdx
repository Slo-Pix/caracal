Use this page only when you are developing Caracal itself. Product users should follow [Quickstart: Released Product](./quickstart/) so they use installed tools and released artifacts.

## Prerequisites

| Tool | Required version |
| --- | --- |
| Docker | 24+ with Compose v2 |
| Node.js | 24+ |
| pnpm | 11.1.1 |
| Bun | 1.3.14 when building distributable runtime or Console binaries |
| Go | 1.26 when running Go tests or changing Go services/packages |
| Python | 3.14 when running Python tests or changing Python packages |

## Install dependencies

```bash
corepack enable
corepack prepare pnpm@11.1.1 --activate
pnpm install --frozen-lockfile --prefer-offline
```

## Start the source-tree stack

```bash
pnpm caracal up
pnpm caracal status --ready
pnpm caracal-console
```

`pnpm caracal` runs the workspace runtime shim. `pnpm caracal-console` runs the workspace Console shim. These commands mirror the released `caracal` and `caracal-console` tools but resolve from the repository.

## Create product objects

Inside the Console, choose **guided setup**. The source-tree flow creates the same real objects as the released flow:

- zone
- confidential agent app
- provider, when needed
- protected resource
- active starter policy
- runtime profile and secret-file instructions
- first-success and audit explanation commands

Set `CARACAL_CONFIG` to the generated profile path before running source-tree workloads:

```bash
CARACAL_CONFIG=/path/to/caracal.toml pnpm caracal run -- ./start-agent.sh
```

## Stop the stack

```bash
pnpm caracal down
```
