Echo Upstream is a zero-dependency HTTP service under `examples/echoUpstream`. It gives the first integration path a local protected target, so you can verify a Gateway-mediated request without hosting your own upstream.

## What it demonstrates

| Area | Behavior |
| --- | --- |
| Upstream target | Returns JSON with the forwarded method, path, headers, and body. |
| Gateway reachability | Joins the `caracalData` Docker network so Gateway can reach `http://echoUpstream:8088`. |
| Local debugging | Exposes `http://127.0.0.1:8088` on the host. |
| Runtime dependency | Runs independently from the Caracal stack. |

## Run with Docker

```bash
cd examples/echoUpstream
docker compose -f compose.yml up --build
```

Check the local health endpoint:

```bash
curl http://127.0.0.1:8088/healthz
```

## Run with Node

```bash
cd examples/echoUpstream
npm start
```

The server listens on port `8088`. Set `ECHO_PORT` when you need a different local port.

## Use as an upstream

In Console or through the Control API, create a resource with this upstream URL:

```text
http://echoUpstream:8088
```

A Gateway-mediated request returns the echoed JSON payload, proving the request passed through Caracal policy enforcement and reached the protected target.
