Caracal CLI commands use a small exit-status surface: success is `0`, command or API failures are `1`, and wrapper commands may return the status of the process they launched. Use stderr and, where available, `--json` output to distinguish authentication, authorization, validation, not-found, conflict, rate-limit, and server errors.

## Convention

| Code | Meaning |
| --- | --- |
| `0` | Success. The operation completed, a help/version request was printed, or every health probe passed. |
| `1` | Command failure. This includes usage errors, missing configuration, failed authentication, failed authorization, not-found objects, validation errors, conflicts, rate limits, service errors, and failed health probes. |
| `127` | `caracal terminal` or `caracal terminal` could not find the sibling binary, or `caracal run` could not spawn the requested subprocess. |
| `<child code>` | `caracal run`, `caracal up`, `caracal down`, and sibling dispatch return the child process or Docker Compose status after the subprocess starts. |
| `128 + N` | A dispatched sibling process ended because of signal `N`. |

`caracal doctor`, `caracal doctor --ready`, `caracal status`, and `caracal status --ready` exit `0` only when every probe passes. Any failed, down, or unreachable dependency exits `1`.

## API error handling

Admin, runtime, and observability commands do not map HTTP statuses to separate exit codes. They exit `1` and print the API error code, HTTP status, remediation hint, and request ID when available.

| HTTP status | Exit status | CLI output |
| --- | --- | --- |
| `401` | `1` | Check `CARACAL_ADMIN_TOKEN`, `app_client_secret`, or `CARACAL_COORDINATOR_TOKEN`. |
| `403` | `1` | Current credentials lack the required scope or zone access. |
| `404` | `1` | Confirm the referenced object ID with the matching `list` command. |
| `409` | `1` | Re-fetch the object and retry the operation. |
| `422` | `1` | Inspect the validation body and correct the submitted fields. |
| `429` | `1` | Back off and retry; inspect service health if throttling persists. |
| `5xx` | `1` | Probe readiness with `caracal doctor` and check service logs. |

## Examples

```bash
# Gate a Helm post-install hook on full readiness:
caracal doctor --ready --json > /tmp/doctor.json || exit 1

# Detect a missing zone in CI:
if ! caracal zone get prod > /tmp/zone.json 2> /tmp/zone.err; then
  if grep -q 'HTTP 404' /tmp/zone.err; then
    caracal zone create --name prod
  else
    cat /tmp/zone.err >&2
    exit 1
  fi
fi

# Preserve a subprocess failure from caracal run:
caracal run -- ./agent-smoke-test.sh
rc=$?
if [ "$rc" -ne 0 ]; then
  echo "agent smoke test failed with status $rc" >&2
  exit "$rc"
fi
```

## Notes

- Usage errors and missing `caracal.toml` both exit `1`; the usage or configuration message on stderr identifies the cause.
- `caracal credential inspect` is local-only and exits `1` for invalid input.
