This page covers what you need installed and how to bring up a local Caracal stack ready for development.

---

## Required toolchains

| Tool | Version | Used for |
| --- | --- | --- |
| Node.js | 24+ | TypeScript services and packages |
| pnpm | 11.0.9+ | TypeScript dependency management |
| Go | 1.26+ | Go services and packages |
| Python | 3.11+ | Python SDK packages |
| Bun | 1.x | CLI/TUI binary compilation |
| Docker Desktop | latest stable | Local service stack |

Install each from its official source. The repository does not manage toolchain versions through scripts — use your system package manager or version manager.

### Go workspace

The repository uses a `go.work` file at the repository root to span all 11 Go modules. After installing Go 1.26+, no additional configuration is needed — `go` commands run from the repo root pick up all modules automatically.

```sh
go work sync   # sync workspace module graph
```

### Python environments

Each Python package has its own virtual environment. Create it before working on that package:

```sh
cd packages/caracalai-python
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

## Clone and install

```sh
git clone https://github.com/garudex-labs/caracal.git
cd caracal
pnpm install
```

`pnpm install` resolves all TypeScript workspace dependencies across `apps/`, `services/`, `packages/`, and `docs/`. Go and Python dependencies are managed separately per module.

## Environment configuration

```sh
pnpm secrets:init
```

`pnpm secrets:init` is the single source of truth for stack credentials. It:

1. Creates `infra/docker/.env` from `infra/docker/.env.example` if missing.
2. Generates random secret files in `infra/secrets/files/` (gitignored, `0444`) for Postgres, Redis, the admin token, zone KEK, and HMAC keys.
3. Writes the same values into `infra/docker/.env` so env-interpolated services and secret-mounted services agree on every credential.

Idempotent and safe to re-run — existing secret files are reused and re-synced into `.env`.

Optional knobs (resource limits, S3 export, etc.) are documented inline in `infra/docker/.env.example` and can be edited freely after `secrets:init`.

Do not commit `infra/docker/.env` or `infra/secrets/files/` — both are in `.gitignore`.

## Start the stack

```sh
caracal up
```

This runs `docker compose up -d` using the compose file in `infra/docker/`. It starts seven services: API (3000), STS (8080), Gateway (8081), Coordinator (4000), Audit (9090), plus PostgreSQL and Redis.

After the stack is up, run the bootstrap initialization:

```sh
caracal init
```

`caracal init` calls the API's bootstrap endpoint, reads the admin token from `infra/docker/.env` via `CARACAL_REPO_ROOT` discovery, and creates the default zone. It writes `caracal.toml` with the zone ID, application credentials, and a default resource. Pass `--admin-token` or set `CARACAL_ADMIN_TOKEN` in your environment to override.

Verify the stack is healthy:

```sh
caracal status
```

All seven services should show `healthy`.

## Database migrations

Migrations run automatically when the API service starts. Migration files live in `infra/postgres/migrations/` as numbered SQL files:

```
NNNN_description.up.sql
NNNN_description.down.sql
```

If you need to run migrations manually against a local Postgres instance:

```sh
psql $DATABASE_URL -f infra/postgres/migrations/0001_init.up.sql
```

The API uses an advisory lock to serialize migration runs across concurrent replicas, so it is safe to start multiple replicas simultaneously.

## Build individual services

**TypeScript services:**

```sh
pnpm --filter @caracalai/api build
pnpm --filter @caracalai/coordinator build
```

**Go services:**

```sh
cd services/sts && go build ./...
cd services/gateway && go build ./...
cd services/audit && go build ./...
```

**CLI/TUI binaries:**

```sh
pnpm --filter @caracalai/cli build
pnpm --filter @caracalai/tui build
# Binaries land in apps/cli/dist/caracal-cli-<os>-<arch> and apps/tui/dist/caracal-tui-<os>-<arch>
```

## Editor setup

The repository has no enforced editor configuration. Recommended plugins:

- **TypeScript**: the built-in TS Language Server via your editor's LSP support.
- **Go**: `gopls` (bundled with Go or installable via `go install golang.org/x/tools/gopls@latest`).
- **Python**: Pylance or Pyright.
- **Rego**: OPA extension for VS Code provides syntax highlighting and `opa check` integration.
