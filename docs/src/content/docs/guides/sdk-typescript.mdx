Use `@caracalai/sdk` in Node applications that need agent sessions, delegation, Gateway routing, or Caracal context propagation.

## Install

```bash
npm install @caracalai/sdk
```

## Configure

The SDK resolves configuration in this order:

1. `Caracal.connect({ clientSecret: ... })` explicit options.
2. `Caracal.connect({ configPath })`.
3. `CARACAL_CONFIG` or the default generated profile.
4. Environment variables.

The Console guided setup can write the profile and secret file for you.

## Connect and spawn

```ts
import { Caracal } from "@caracalai/sdk";

const caracal = Caracal.connect();

await caracal.spawn(async () => {
  const headers = await caracal.headersAsync();

  await fetch("https://api.example.com/tickets", {
    headers,
  });
});
```

`spawn()` creates an agent session, binds the Caracal context while the callback runs, and terminates non-service sessions when the callback exits.

To make agents distinguishable in policy and audit, pass `kind` and `capabilities`. These become `input.principal.agent_kind` and `input.principal.capabilities`, so several agents under one application stay separable without one application per agent.

```ts
await caracal.spawn(async () => {
  const headers = await caracal.headersAsync();
  await fetch("https://api.example.com/tickets", { headers });
}, { kind: "instance", capabilities: ["refunds.execute"] });
```

See [If many agents share one managed application, can policy and audit still tell them apart?](/reference/faq/#faq-distinguish-agents-under-one-application).

## Delegate to a child agent

```ts
await caracal.spawn(async () => {
  await caracal.delegateToSpawn({
    resourceId: "https://api.example.com/tickets",
    scopes: ["tickets:read"],
    constraints: { maxHops: 1, budget: 5 },
    delegationTtlSeconds: 600,
  }, async () => {
    const headers = await caracal.headersAsync();
    await fetch("https://api.example.com/tickets", { headers });
  });
});
```

Use `delegate()` when the target agent session already exists. Use `delegateToSpawn()` when fan-out should create the child session and delegation edge together.

## Route through the Gateway

```ts
const request = caracal.gatewayRequest("https://api.example.com/tickets", "/tickets");

await fetch(request.url, {
  headers: {
    ...request.headers,
    ...(await caracal.headersAsync({ allowRoot: true })),
  },
});
```

For provider SDKs that accept a custom `fetch`, pass `caracal.transport()` so Caracal headers and Gateway routing are applied automatically.

## Troubleshooting

| Symptom | Check |
| --- | --- |
| `Caracal.fromEnv: missing ...` | Confirm generated profile or required `CARACAL_*` variables are present. |
| Root headers rejected | Call `headersAsync({ allowRoot: true })` only when service-root identity is intentional. |
| Delegation fails | Ensure the call runs inside `spawn()` or another bound context. |
| Gateway request misses resource | Confirm `gateway_url`, resource bindings, and `X-Caracal-Resource`. |
