---
title: Runtime Lifecycle
description: Use caracal up, down, status, and purge to manage the local runtime stack.
pageType: reference
---

Runtime lifecycle commands manage local infrastructure and do not perform product management. They delegate to Docker Compose, resolve release assets from the Caracal home, and preserve the terminal management interface as the single human management surface.

---

## `caracal up`

Start the stack:

```bash
caracal up
```

In development mode, the command rebuilds local images. In rc and stable modes, it uses the installed release compose assets and injects `CARACAL_VERSION`.

---

## `caracal down`

Stop stack containers:

```bash
caracal down
caracal down -v
```

`-v` is passed to Docker Compose and removes persisted volumes.

---

## `caracal status`

Probe service liveness or dependency-aware readiness:

```bash
caracal status
caracal status --ready
caracal status --ready --json
```

Readiness checks include Postgres, Redis, STS reachability, Gateway bindings, audit replay, and stream consumers. The command exits `0` when every probe passes and `1` when any service is down, unreachable, or not ready.

---

## `caracal purge`

Remove the local stack, runtime data, generated logs, cached release assets, and installed runtime state:

```bash
caracal purge
```

Use `purge` only when you intentionally want a clean local environment. Recreate zones, applications, policies, grants, and Control keys through the terminal management interface after the next `caracal up`.
