Stack commands manage the local Caracal stack. They delegate to `docker compose` and do not require `CARACAL_ADMIN_TOKEN`. In rc or stable mode, the compose file and env file are resolved automatically from the Caracal home — no flags needed. In dev mode, run via `pnpm caracal` from inside the repo, which sets `CARACAL_REPO_ROOT` automatically.

---

## `caracal up`

Start the stack.

```bash
caracal up
```

Equivalent to `docker compose up -d`. In dev mode, passes `--build` to rebuild images before starting. In rc or stable mode, runs without `--build` and injects `CARACAL_VERSION` into the compose environment.

Forwards all `docker compose` output to stdout and exits with the compose exit code.

---

## `caracal down`

Stop and remove stack containers.

```bash
caracal down       # stop containers, preserve volumes
caracal down -v    # stop containers and remove volumes (destroys all persisted data)
```

`-v` is passed directly to `docker compose down`. Use it to reset the stack to a clean state between test runs or when rotating secrets.

## `caracal status`

Probe all five services and report their liveness or readiness.

```bash
caracal status
caracal status --ready
caracal status --ready --json
```

By default, sends `GET /health` to each service in parallel with a 1500 ms timeout. Use `--ready` to send `GET /ready`, which checks dependencies and service-specific readiness such as policy consumers, Gateway bindings, audit replay, Postgres, and Redis. When a readiness endpoint returns structured JSON, the `detail` column includes the reason so operators can distinguish a network outage from a dependency or replay condition.

| Service | Default port | Health URL | Readiness URL |
| --- | --- | --- | --- |
| `api` | 3000 | `/health` | `/ready` |
| `sts` | 8080 | `/health` | `/ready` |
| `gateway` | 8081 | `/health` | `/ready` |
| `audit` | 9090 | `/health` | `/ready` |
| `coordinator` | 4000 | `/health` | `/ready` |

Example output:

```
service      port   mode    status  detail
-------      ----   ----    ------  ------
api          3000   ready   ok      200
sts          8080   ready   down    503 stream_consumers_not_ready
gateway      8081   ready   ok      200
audit        9090   ready   ok      200
coordinator  4000   ready   ok      200
```

Exit code is `0` if all probes respond successfully, `1` if any is down or unreachable. Use readiness mode in scripts that need dependency-aware startup:

```bash
caracal up
until caracal status --ready; do sleep 1; done
```

## Provisioning a zone and application

The stack starts with an empty database. Use the admin commands to create a zone, register an application, capture the client secret, and author `caracal.toml`:

```bash
caracal zone create --name local
caracal app create --zone <zone-id> --name local-app
# client secret is printed once; save it into caracal.toml
```

See the [Admin commands](./admin/) reference for full flags and the [Configuration File](./config-file/) page for the `caracal.toml` schema and discovery order.
