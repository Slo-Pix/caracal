Operational observability is available through the terminal management interface. Open it with `caracal terminal` or `caracal-terminal`, then use the diagnostics and audit views for interactive triage.

---

## Diagnostics

Use **Doctor** to inspect service readiness, configured endpoints, selected zone health, resource and grant visibility, policy-set status, audit queryability, and metric summaries. The view uses the same engine checks as runtime readiness and presents failures with remediation hints.

---

## Audit events

Use **Audit** to browse recent events, filter by decision or event type, and inspect the records generated by STS, Gateway, Coordinator, and policy evaluation. Denials include the determining policy metadata and diagnostics needed for incident notes.

---

## Explanations

Open an audit event to inspect the full decision context, including request identifiers, authority anchors, selected resource, provider mode, policy results, and diagnostic payloads. Use this view when a Gateway or STS response needs to be tied back to a policy decision.

---

## Token inspection

Use the credential inspection view to decode a token locally and compare its issuer, subject, zone, session, target resource, scopes, expiry, algorithm, key ID, and token ID with the audit trail. Local decoding is an inspection aid and does not replace signature verification by services.
