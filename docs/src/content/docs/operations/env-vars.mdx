Every Caracal service is configured entirely through environment variables. This page lists all variables grouped by service. Variables marked **required** will cause the service to fail at startup if absent. Variables with defaults are optional.

## Shared secrets

Manual environment-driven deployments must provide the shared secrets required by the services they run before startup. Generate each with `openssl rand -hex 32`.

The installed local stack created by `caracal up` generates file-backed equivalents under `$CARACAL_HOME/secrets/` and passes them to services through the matching `*_FILE` variables.

For service deployments, prefer the `*_FILE` form for sensitive values so the secret is mounted from a secret store instead of placed directly in the process environment.

| Variable | Services that use it | Description |
| --- | --- | --- |
| `ZONE_KEK` | STS, API | 32-byte hex-encoded key encryption key. Encrypts zone signing key material at rest. Must not be all zeros. |
| `AUDIT_HMAC_KEY` | STS, Audit, Gateway, Control | 32-byte hex-encoded key for audit event chain HMAC. Required when `CARACAL_MODE=rc` or `CARACAL_MODE=stable` on services that verify or emit protected audit events. |
| `STREAMS_HMAC_KEY` | STS, API, Gateway, Coordinator, coordinator relay | 32-byte hex-encoded key for signing or verifying Redis stream messages. Required by Gateway; required by STS, API, Coordinator, and relay when `CARACAL_MODE=rc` or `CARACAL_MODE=stable`. |
| `GATEWAY_STS_HMAC_KEY` | STS, Gateway, API | 32-byte hex-encoded key for signed internal Gateway/API calls to STS. Required by STS and Gateway when `CARACAL_MODE=rc` or `CARACAL_MODE=stable`; optional on API for signed policy simulation. |

## Mode

| Variable | Default | Description |
| --- | --- | --- |
| `CARACAL_MODE` | `stable` in service code; `dev` in local Docker Compose | `dev`, `rc`, or `stable`. rc and stable enable startup safety checks and require signed stream/audit keys on protected paths. |

## Postgres

| Variable | Default | Description |
| --- | --- | --- |
| `POSTGRES_USER` | `caracal` | Database user |
| `POSTGRES_PASSWORD` | None | **Required.** Database password |
| `POSTGRES_DB` | `caracal` | Database name |

Application services read `DATABASE_URL`. The Docker/engine tooling uses the individual `POSTGRES_*` variables to initialize Postgres and generate the `DATABASE_URL` secret mounted into services.

## Redis

| Variable | Default | Description |
| --- | --- | --- |
| `REDIS_PASSWORD` | None | **Required.** Redis auth password |
| `REDIS_HOST` | `localhost` | Redis host (used when REDIS_URL is absent) |
| `REDIS_PORT` | `6379` | Redis port |

Application services read `REDIS_URL`. Redis provisioning scripts use `REDIS_HOST`, `REDIS_PORT`, and `REDIS_PASSWORD` when creating streams and consumer groups.

## Optional Control service

The control service is unmounted by default and managed through the authenticated interactive Console Control flow or the Console Control menu. The variables below are consumed by the managed service after the engine starts the optional Control runtime.

| Variable | Default | Description |
| --- | --- | --- |
| `CARACAL_CONTROL_ENABLED` | managed by the runtime | Must be `true` inside the Control service; the runtime/console lifecycle sets this through the managed runtime definition |
| `CONTROL_GATE_FILE` | `/run/caracal-control/enabled` | Local gate file checked by the Control service before readiness and invoke requests |
| `CARACAL_CONTROL_STATE_DIR` | managed by the runtime | Host directory mounted read-only into the Control container so runtime/console can open or close the endpoint gate |
| `CONTROL_PORT` | `8087` | Listen port |
| `CONTROL_HOST` | `127.0.0.1` (dev), `0.0.0.0` (rc/stable) | Listen address |
| `STS_JWKS_URL` | None | **Required.** STS JWKS URL used to verify bearer tokens (`?zone_id=…` is appended per token) |
| `STS_ISSUER_URL` | None | **Required.** Expected issuer for control bearer tokens |
| `CONTROL_AUDIENCE` | None | **Required.** Expected audience for control bearer tokens |
| `CARACAL_API_URL` | None | **Required.** Admin API base URL the engine dispatches to (e.g. `http://api:3000`) |
| `CONTROL_API_TOKEN` / `CONTROL_API_TOKEN_FILE` | None | **Required.** Admin token used by the engine when calling the API. Prefer the file-backed form. |
| `CONTROL_REDIS_URL` | None | Replay cache and audit sink Redis URL. Required when `CARACAL_MODE=rc` or `CARACAL_MODE=stable`; optional in dev mode |
| `AUDIT_HMAC_KEY` | None | Hex-encoded ≥32-byte key for HMAC-SHA256 signing of audit rows. Required when `CARACAL_MODE=rc` or `CARACAL_MODE=stable` |
| `CONTROL_RATE_CAPACITY` | `60` | Per-subject token-bucket capacity |
| `CONTROL_RATE_WINDOW_SEC` | `60` | Per-subject token-bucket refill window |
| `CONTROL_REPLAY_TTL_SEC` | `3600` | Maximum JTI replay retention; bounded further by the token's `exp` |
| `LOG_LEVEL` | `info` | Fastify logger level |
