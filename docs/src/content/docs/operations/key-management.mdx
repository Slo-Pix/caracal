Caracal uses a zone key-encryption key (`ZONE_KEK`) to encrypt zone signing keys before they are stored in Postgres. The STS decrypts the active signing key in memory, issues ES256 mandates with its `kid`, and publishes the active key plus the prior rotation key through JWKS while the grace window is open.

## Required key material

| Key | Owner | Requirement |
| --- | --- | --- |
| `ZONE_KEK` | Platform secret manager | 32-byte hex key. Required for STS and services that encrypt or decrypt zone secrets. |
| `AUDIT_HMAC_KEY` | Platform secret manager | 32-byte hex key shared by STS and Audit for audit event integrity. |
| `STREAMS_HMAC_KEY` | Platform secret manager | 32-byte hex key for Redis stream origin verification. |
| `GATEWAY_STS_HMAC_KEY` | Platform secret manager | 32-byte hex key for signed Gateway/API calls into STS. |
| `STS_ADMIN_TOKEN` | Platform secret manager | Bearer token for STS operator endpoints. When unset, operator endpoints are disabled. Helm deployments should mount it from `secrets.stsAdminToken.secretName`, separate from `CARACAL_ADMIN_TOKEN`. |

Use the file-backed variables (`*_FILE`) in production so key material is mounted from the platform secret manager and never embedded in manifests.

## Zone signing-key rotation

Rotate a zone signing key from a trusted operator network path:

```bash
curl -X POST \
  -H "Authorization: Bearer ${STS_ADMIN_TOKEN}" \
  "https://sts.example.internal/internal/zones/${ZONE_ID}/signing-key/rotate"
```

The response returns the new `kid`. The STS immediately signs new mandates with the rotated key and invalidates its local key caches. JWKS continues to publish the previous key during the 24-hour grace window so already-issued mandates remain verifiable until they expire.

After rotation:

1. Confirm `/.well-known/jwks.json?zone_id=${ZONE_ID}` contains the new `kid`.
2. Confirm new token exchanges return mandates whose JWT header uses the new `kid`.
3. Keep the previous key in Postgres through the grace window; do not delete historical key rows manually.

## KEK rotation boundary

`ZONE_KEK_PROVIDER=local` is the supported provider in this repository. Rotating `ZONE_KEK` requires a controlled re-encryption workflow for stored zone secrets before the old key is removed. Do not change `ZONE_KEK` in-place on running services without re-encrypting existing rows, because the STS will fail closed when it cannot decrypt signing keys.
