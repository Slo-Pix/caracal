Caracal has two local operator surfaces with a deliberate boundary:

| Surface | Owns | Does not own |
| --- | --- | --- |
| `caracal` | Local stack lifecycle, readiness, local reset, token-injected process execution, and Console launch. | Product-management workflows. |
| `caracal console` / `caracal-console` | Human management for zones, applications, providers, resources, policies, policy sets, grants, sessions, Control, audit, explanations, agent runs, delegation, and diagnostics. | Starting/stopping the stack or running workloads. |

The runtime CLI stays small so local lifecycle and workload execution do not depend on admin tokens, zone selection, or Control credentials. Product management belongs to the Console or to the authenticated Control/Admin APIs.

## Runtime map

| Command | Purpose |
| --- | --- |
| `caracal up` | Start the local Docker Compose stack. |
| `caracal down` | Stop the stack; pass `-v` to remove volumes. |
| `caracal status` | Probe service health. |
| `caracal status --ready` | Probe dependency readiness. |
| `caracal purge` | Remove stack artifacts and selected local state. |
| `caracal run -- <command>` | Run a subprocess with scoped resource tokens injected. |
| `caracal console` | Launch the Console when the Console binary or workspace shim is available. |

No other top-level `caracal` product-management commands are part of the runtime CLI.

## Console map

| Menu label | Key | Purpose |
| --- | --- | --- |
| `guided setup` | `s` | Create the first zone, agent app, provider, resource, policy, and runtime profile. |
| `zone` | `1` | Manage zones. |
| `application` | `2` | Manage confidential agent applications. |
| `provider` | `3` | Manage provider credential sources. |
| `resource` | `4` | Manage protected resources. |
| `policy` | `5` | Manage access policies. |
| `policy set` | `6` | Manage active policy sets. |
| `grant` | `7` | Manage access grants. |
| `session` | `8` | List active authority sessions. |
| `control` | `9` | Manage the Control automation service and credentials. |
| `audit` | `a` | Search audit events. |
| `explain` | `e` | Explain an audit decision. |
| `agent run` | `r` | Manage agent runs. |
| `delegation` | `g` | Manage delegated permissions. |
| `diagnostics` | `d` | Run operator diagnostics. |

## Learning path

1. Start with [CLI and Console](/runtime-console/cli-and-console/) to choose the right surface.
2. Use [Runtime Lifecycle](/runtime-console/stack/) to start and check the local stack.
3. Run [Console guided setup](/runtime-console/console/) to create a real runtime profile.
4. Use [Configuration File](/runtime-console/config-file/) and [Runtime Execution](/runtime-console/runtime/) when running workloads.
5. Use [Management Workflows](/runtime-console/admin/), [Observability Workflows](/runtime-console/observability/), and [Agents and Delegation](/runtime-console/agents/) for day-to-day operations.
