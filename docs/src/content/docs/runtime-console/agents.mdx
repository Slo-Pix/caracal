Agent and delegation operations are available from the Console. In the local stack, Console prefers the Coordinator token generated by `caracal up`, even if the shell still has an older exported token. Set `CARACAL_COORDINATOR_URL` or `CARACAL_COORDINATOR_TOKEN` only when pointing Console at a non-default Coordinator.

---

## Agent sessions

Use **Agents** to list sessions, inspect a session tree, suspend an active subtree, resume a suspended subtree, or terminate a session. Tree views show parent-child relationships so operators can understand which descendants inherit authority from the selected run.

---

## Delegation edges

Use **Delegation** to inspect active, inbound, and outbound delegation edges. The detail view shows source and target sessions, resource and scope bounds, expiry, and traversal state.

Revoking a delegation edge terminates its delegated authority immediately. Review outbound edges before terminating a session so dependent work can be stopped cleanly.

---

## Audit correlation

Agent and delegation identifiers also appear in audit events. Open the matching audit record when you need the policy decision, requested resource, provider mode, and diagnostics that caused an allow or deny.
