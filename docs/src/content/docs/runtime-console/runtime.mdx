Runtime execution is intentionally limited to the top-level `caracal run` command. Product management belongs to the Console or the Control API.

---

## `caracal run`

Inject scoped tokens into a subprocess environment and run it:

```bash
caracal run -- <command> [args...]
```

The `--` separator is optional but recommended when the subprocess accepts flags. Arguments after `--` are passed verbatim to the subprocess.

### What it does

1. Loads validated runtime config from environment variables or `caracal.toml`.
2. Checks MCP governance rules.
3. Exchanges each required `credentials[]` entry for a scoped Caracal access token.
4. Attempts each `optional_credentials[]` entry and follows its configured failure behavior.
5. Spawns the subprocess with the resulting tokens injected as named environment variables.
6. Forwards stdin, stdout, and stderr unchanged.
7. Exits with the subprocess exit code.

Each credential exchange posts to `{zone_url}/oauth/2/token` using `zone_id`, `application_id`, and the confidential application secret from the validated runtime config. Tokens are injected under the declared `env` names.

The runtime first resolves platform configuration from `CARACAL_STS_URL`, `CARACAL_ZONE_ID`, `CARACAL_APPLICATION_ID`, `CARACAL_APP_CLIENT_SECRET_FILE`, and `CARACAL_RUN_CREDENTIALS_FILE`. TOML remains available through `CARACAL_CONFIG` or the XDG runtime profile path for tools that need a readable file. It does not load project-local files from the current working directory because local config may contain credential material, and Console/admin commands do not load runtime profiles.

### Minimal configuration

```toml
zone_url = "http://localhost:8080"
zone_id = "zone_abc123"
application_id = "app_def456"
app_client_secret_file = "/run/secrets/caracal-app-client-secret"

[[credentials]]
env = "RESOURCE_TOKEN"
resource = "resource://my-api"
```

```bash
caracal run -- python3 agent.py
```

### Step-up challenges

If the STS returns `interaction_required`, `caracal run` writes a JSON notice to stderr, polls the step-up endpoint for satisfaction, and retries the exchange until the challenge succeeds or the runtime window expires.

### MCP governance

`caracal run` detects subprocess commands that reference MCP server binaries. Configure the behavior in `caracal.toml`:

```toml
[mcp_governance]
mode = "block"   # "block" or "log"
```

`"block"` exits before spawning the subprocess. `"log"` writes the notice and continues.
Outside development, `"log"` requires an explicit platform override so runtime profiles cannot silently weaken enforcement.
