The Console is the only supported human-facing surface for Caracal product management. Launch it with:

```bash
caracal console
```

or:

```bash
caracal-console
```

---

## Zones and applications

Use **Zones** to create, inspect, select, rename, and delete zones. Use **Applications** inside the selected zone to register applications, generate confidential client secrets, update consent settings, and remove applications.

Client secrets are shown once when generated. Store them in `caracal.toml` or the appropriate secret manager before leaving the result view.

---

## Resources and providers

Use **Resources** to register protected resources, scopes, upstream URLs, and Gateway application bindings. Use **Providers** to configure direct credential material, external provider plugins, or OIDC-backed provider credentials.

Secret provider fields are masked in the interface and in error output.

---

## Policies and policy sets

Use **Policies** to create policy versions from Rego content, inspect existing versions, and remove obsolete policies. Use **Policy sets** to assemble policy versions, simulate candidate inputs, activate validated versions, and configure shadow evaluation.

Activation is a management mutation and should be performed only after reviewing simulation output and any expected audit impact.

---

## Grants and sessions

Use **Grants** to create and revoke resource access for applications and subjects. Use **Sessions** to inspect active and expired authority. Session records are read-only from the Console; revoke access by revoking the underlying grant or Control key.

---

## Control API handoff

When management must be automated, enable and configure the Control API from the Console. Control API requests are authenticated, audited, and dispatched through the same engine management path as interactive operations.
