# apps/web

## Scope

- Covers the `@caracalai/web` React single-page application under `apps/web/`.
- Serves the marketing surface and the browser-based web console, the primary management interface for Caracal.

## Architecture Design

- `src/main.tsx` mounts the SPA; `src/router.tsx` builds the type-safe TanStack Router instance.
- `src/routes/` owns file-based routes; `src/routeTree.gen.ts` is generated by the TanStack Router Vite plugin.
- `src/routes/app.*` own the authenticated Console; `sign-in`, `sign-up`, `reset`, and `onboarding` own the entry flow.
- `src/platform/` owns edition-agnostic seams: `config`, `auth` boundary, `content`, `nav`, `edition`, and the temporary `state` store.
- `src/editions/community/` owns Community Edition implementations, including the Better Auth client.
- `src/components/` owns shared `ui`, `console`, `auth`, and `onboarding` components.
- `src/styles/globals.css` owns the Tailwind theme and global styles.
- `index.html` owns the document shell and the pre-paint theme bootstrap.

## Required

- Must run on Node 22+ and build to static assets with `vite build`.
- Must keep client-only rendering: no server-side rendering runtime, no Nitro, no server entry.
- Must reach Caracal services only over HTTP through `@caracalai/sdk` or the Control API, never by importing `@caracalai/admin`, `@caracalai/engine`, or `@caracalai/core` source.
- Must reach authentication only over HTTP through the `@/platform/auth` boundary, never by importing identity internals directly into pages.
- Must keep session and provider probes resilient so a failed auth backend never breaks public or auth pages.
- Must consume enterprise capabilities only as locked surfaces driven by static content, never by embedding enterprise logic.
- Must keep per-route metadata working through the route `head()` API and `<HeadContent />`.
- Must keep the dev and preview servers on the dedicated local port.

## Forbidden

- Must not reintroduce SSR, hydration, or a server runtime.
- Must not embed secrets, tokens, or provider credentials in client code.
- Must not import source across the product boundary or from other workspace packages' internals.

## Validation

- Validate with `pnpm --dir apps/web typecheck`, `pnpm --dir apps/web lint`, and `pnpm --dir apps/web build` when web code changes.
