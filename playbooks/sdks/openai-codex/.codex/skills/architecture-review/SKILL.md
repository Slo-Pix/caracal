# Architecture Review

Use to review framework, runtime, structure, middleware, services, configuration, and integration points before Caracal SDK work.

## Procedure

1. Determine language, framework, runtime, package manager, and deployment model.
2. Map project structure, service boundaries, middleware, routing, dependency injection, and configuration patterns.
3. Find existing auth and authorization flow.
4. Identify where Caracal naturally fits and where it should not be introduced.
5. Prefer existing files and patterns over new directories or abstractions.

Return recommended and optional integration points with reasons.
